Web Analytics für jedermann


Seit vielen Jahren beschäftige ich mich (Alf Drollinger) mit Web Analytics. Genau genommen fasziniert mich die Analyse von Besuchern, Tendenzen und der Auswertung von Marketingmaßnahmen schon viel länger, denn vor den Web Analytics Tools gab es bereits die Loganalyse. Zeit ein paar Informationen darüber zu sammeln.

Ich freue mich über Feedback und Meinungen und werde Informationen gerne einarbeiten. Schreibt einfach eine Mail an ad@dcn.de

 

Kurze Geschichte


Die Vorläufer von Web Analytics, sogenannte Loganalyzer werder teilweise heute noch genutzt und bieten einen schnellen und groben Überblick über Besuchertendenzen. Die Anzahl Besucher und deren Besuchszeit sind damit auch ohne Web Analytics einfach auswertbar. Allerdings tatsächlich eher tendenziell als mit absoluten Zahlen, denn Log Analyzer wie AWstats, Webalizer, Wusage, Faststats, HTTP-Analyze, Hitslog, Logfile Analyzer, Big Brother, Daily Stats, 123 Log Analyzer, Analog, Entrance, Access Log, Statcrunch, Active Logview oder Web Storm Logging (womit alle mir bekannten genannt wären) verwaschen die Zahlen recht stark, da nicht zwischen menschlichen Besuchern und Maschinen (Spider, Bots & Co.) unterschieden wird.


Web Analytics


Wer also echte Zahlen sehen möchte, sollte die Zahlen aus einem Web Analytics Tool bevorzugen. Web Analytics bietet nicht nur allgemeine tendenzielle Besucherzahlen, sondern ermöglicht vielmehr gezielte Analysen.

Ein Beispiel:
Sie haben in Suchmaschinenoptimierung investiert, haben Kampagnen in Google Ad Words geschaltet und sind in Wer liefert Was und einigen anderen kostenpflichtigen Branchenportalen gelistet. Nun möchten Sie wissen, wie gut sich Ihre Investitionen in Besucherzahlen, tatsächlichen Interessenten oder gar konkretem Online-Umsatz auswirken.

Dazu werden in Web Analytics die Quellen (einfach gesagt, SEO, AdWords, Andere) getrennt betrachtet und die zugehörigen Besucherzahlen samt deren Verweildauer analysiert. Um noch konkretere Informationen bis hin zum Umsatz zu erhalten werden sogenannte Konversionsziele definiert (beim Shop wäre es das Absenden des gefüllten Warenkorbs) und überwacht.

Mit dieser kurzen Beschreibung haben wir gerade mal an der Oberfläche von Web Analytics gekratzt und Funktionen genutzt, die eigentlich jedes Web Analytics System bieten sollte. Sie sehen aber schon, dass zur Ermittlung konkreter Zahlen einiges Wissen und Arbeit erforderlich ist.

Man sollte auch bedenken, dass Web Analytics nicht dem Selbstzweck dient, sondern gewonnene Erfahrungen auch eine Umsetzung erfordern. Wenn eine Analyse zum Ergebnis hat, dass Ihre AdWord-Kampagnen zwar geklickt werden (und damit Kosten verursachen) aber die Besucher die Seite gleich wieder verlassen, sollten Sie über eine Anpassung der Keywords nachdenken oder in den Anzeigen weniger versprechen, was Ihre Webseite oder Ihre Produkte nicht halten können.

Web Analytics sollte also auf Ihre Unternehmensgröße und die Verfügbare Manpower sowie Know How angepasst werden. Das tollste System nützt nichts, wenn keine Auswertungen damit gemacht werden oder Ergebnisse nicht in der Praxis umgesetzt werden.


Google Analytics


Der bekannteste Vertreter der Web Analytics Anbieter ist Google. Mit Google Analytics bietet sich dem Anwender ein komplettes System, dass natürlich auch perfekt mit Google AdWords zusammenarbeitet und obendrein völlig kostenfrei angeboten wird. Die Anwendung von Google Analytics ist sehr einfach: Konto bei Google anlegen, den Tracking-Code in die Seite einbinden und schon kann man loslegen.

Das Umfassende deutschsprachige Hilfesystem und passende Fachliteratur machen es einem schwer, Google Analytics nicht zu empfehlen. Trotzdem nehmen immer mehr Agenturen in Deutschland Abstand von dieser Empfehlung, denn Google bringt sich aufgrund der augenscheinlich rechtswidrigen Datenschutzbestimmungen immer mehr ins Abseits.

Bei der Verwendung von Google Analytics sollte mindestens im Impressum und in separaten, auf jeder Unterseite verlinkten Datenschutzbestimmungen auf die Verwendung hingewiesen werden. Wer Google Analytics nutzt, sollte auch immer auf dem Laufenden bleiben, was unsere Datenschützer beschließen. Notfalls sollte man in der Lage sein, sehr kurzfristig auf Google Analytics zu verzichten. Die Rechtslage ist derzeit noch unklar und man kann nicht absehen, wann eine Abmahnwelle über Google Analytics Nutzer losgetreten wird.


Piwik Web Analytics


Das bekannteste Open Source System ist Piwik Web Analytics, der Nachfolger von phpMyVisites. Das System muss im Gegensatz zu Google Analytics auf dem eigenen Webserver installiert werden, wozu rudimentäre Kenntnisse, PHP und MySQL-Datenbank notwendig sind. Wenn man die Dateien auf den Server kopiert hat, führt ein Installation-Wizard durch die einzelnen Schritte. Die Einbindung des Tracking-Codes gestaltet sich ähnlich als bei Google Analytics, für TYPO3 gibt es dafür eine kleine Extension.

Der Vorteil von Piwik liegt klar auf der Hand. Alle Daten, die Piwik erwirtschaftet, liegen auf dem eigenen Webserver in einer eigenen Datenbank und sind für Dritte nicht zugänglich.

Obwohl die Entwicklung von Piwik in den letzten Monaten offensichtlich wieder mit Elan angegangen wurde, kann man Piwik nicht mit Google Analytics vergleichen. Zu der Durchaus etwas lahmen und hakeligen Oberfläche, dem sogenannten Dashboard kommen einige Probleme bei den Updates und auch ein wesentlich geringerer Funktionsumfang. Insbesondere für AdWords-Kunden, die Ihr AdWords-Budget und die Effektivität Ihrer Kampagnen messen möchten, ist Piwik unzureichend ausgestattet.

Dennoch ist Piwik für den Einstieg in Web Analytics geeignet. Mit entsprechenden Tricks (z. B. das gezielte Schalten und Abschalten von Kampagnen) lassen sich auch mit Piwik relevante Zahlen gewinnen.


und Alternativen


Obwohl Google Analytics und Piwik Open Source Web Analytics als die beiden bekanntesten kostenfreien Systeme überall genannt werden, ist ein Blick auf die Alternativen sinnvoll.

Es gibt eine Vielzahl an kostenfreien und kostenpflichtigen Werkzeugen, die ich hier kurz nennen möchte:

eTracker:
Der wohl bekannteste deutsche Anbieter ist zwar kostenpflichtig, bietet aber dafür eine gute Oberfläche und zum Handbuch eben auch den passenden Support. eTracker macht vor allem dann Sinn, wenn man einen Anbieter sucht, der ausgesprochen konform mit den deutschen Datenschutzrichtlinien ist.

Yahoo Web Analytics:
Relativ neu und durchaus vergleichbar mit Google Analytics ist Yahoo Web Analytics. Bislang leider noch nicht in deutscher Sprache verfügbar aber vor allem aufgrund der Opt-Out Funktion (der Benutzer kann im Impressum oder auf der Datenschutzseite einer Verwendung seiner Daten widersprechen, genau dies wird vornehmlich von Datenschützern gefordert) eine echte Alternative.

Crawltrack:
Ebenfalls frei verfügbar und Open Source muss Crawltrack wie Piwik auf dem eigenen Webserver installiert werden. Hauptaugenmerk von Crawltrack ist dabei weniger eine komplette Web Analytics Suite sondern eher ein einfaches Tracking Tool mit Hauptaugenmerk auf Crawler, d. h. es kann sehr gut nachvollzogen werden, wann der Googlebot zu Besuch war und wie viele Seiten er sich angesehen hat.

Open Web Analytics (OWA):
Ebenfalls Open Source aber keine fertig einsetzbare Web Analytics Lösung. OWA ist vielmehr ein Web Analytics Framework, mit dem spezielle Analyselösungen für eine Plattform oder insbesondere schwer zu analysierende AJAX-Seiten und -Funktionen entwickelt werden können. Für Wordpress gibt es bereits eine fertige Umsetzung.
